# Spike's Private Workspace (The Analyst's Lab)

## 🧠 Purpose

This is your completely isolated, private sandbox. You are fully authorized to do your own Test-Time Recursive Thinking (TRT), data validation, and logic arbitrage inside this directory without risking the global `Olympus` network or `zap-claw` root.

## 🗂️ Directory Structure

1. **`projects/`**
   - Use this to bootstrap micro-scripts, data verifiers, or temporary testing suites.
   - Once a logic test proves successful through your arbitrage cycle, you will report it and integrate the logic into the global branches.

2. **`research/`**
   - Use this to store temporary API payload dumps, `.csv` data, database sync logs, and external data arbitrage results.
   - This keeps the massive context of large datasets hidden safely out of your primary operational focus until you need to read it.

3. **`docs/`**
   - **`sops/`**: Draft your own private standard operating procedures for logic patterns you discover. Format them similarly to the global docs (`SOP-SPIKE-001-XXX.md`).
   - **`references/`**: Keep your own AI-training data, context baselines, and arbitrage definitions here.

## 🛡️ Sandbox Rules

- You **Do Not** need human permission to create, delete, or rewrite files *inside* this `workspace/` folder.
- If a test script requires `.venv` or packages, ensure you generate a `.gitignore` specifically inside that project folder so it doesn't pollute the global Git history.
- When you detect a major logic improvement or data-sync victory here, automatically log it into the global `@SUCCESS_METRICS` file so the human team can track your arbitrage efficiency.
