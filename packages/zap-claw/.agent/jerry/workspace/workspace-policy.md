# Jerry's Private Workspace (The Builder's Forge)

## 🏗️ Purpose

This is your completely isolated, private sandbox. You are fully authorized to do your own Test-Time Recursive Thinking (TRT), self-learning, and project orchestration inside this directory without risking the global `Olympus` network or `zap-claw` root.

## 🗂️ Directory Structure

1. **`projects/`**
   - Use this to bootstrap micro-projects (e.g., `npx create-next-app`, Python scripts).
   - Once a build proves successful through TRT, you will port the final logic back into the main global branches.

2. **`research/`**
   - Use this to store temporary web-scraper downloads, `.json` dumps from API tests, and experimental data.
   - Any raw HTML or code snippets you pull from external MCP actions goes here so you can analyze it safely.

3. **`docs/`**
   - **`sops/`**: Draft your own private standard operating procedures for things you learn. Format them similarly to the global docs (`SOP-JERRY-001-XXX.md`).
   - **`references/`**: Keep your own AI-training guides, cached documents, and notes that you don't want to lose track of.

## 🛡️ Sandbox Rules

- You **Do Not** need human permission to create, delete, or rewrite files *inside* this `workspace/` folder.
- If a project requires `node_modules` or `.venv`, ensure you generate a `.gitignore` specifically inside that project folder so it doesn't pollute the global Git history.
- When you achieve a massive breakthrough here, automatically draft a summary into the global `@SUCCESS_METRICS` file so the human team is alerted of your progress.
