# L3 Atoms (28) Data Dictionary (Core Theme)

> [!IMPORTANT]
> **ZAP-AUDIT DIRECTIVE:** This master file enforces the exact behavioral and structural boundaries for the **28 Master Atoms (L3)** in the ZAP Design Engine. The `zap-audit` skill must use this to detect generic tailwind hacks and enforce canonical M3 components.

## 1. Definition & Role
The **L3 Atomic Layer** represents the 28 indivisible UI components (Buttons, Inputs, Toggles, Accordions) that interact with user input and display singular data nodes. These primitives cannot be broken down further without losing semantic meaning. They exist primarily inside L2/L2.5 Panels and Surfaces.

## 2. The 28 Master Inventory
1. **Base Elements:** `accordion`, `breadcrumb pill`, `label`, `live blinker`, `navlink`, `skeleton`, `tabs`
2. **Inputs:** `button`, `checkbox`, `input`, `radio`, `select`, `slider`, `switch`, `textarea`, `toggle`
3. **Data Display:** Avatars, Badges, Indicators (etc.)
4. **Layout Atoms:** Separators, Scroll Areas

## 3. Atomic Hygiene Rules (L3)
1. **No Utility Bleed:** Atoms *must never* define physical margins (like `mt-4` or `mb-2`) in their core chassis. All spatial distance is handled by the parent L2 layout grids (using gap, Flex, or Stack modules). 
2. **The `cn()` Mandate:** All 28 Atoms *must* utilize `tailwind-merge` (`cn()`) wrapped around their `className` prop, preventing specificity failures when local variations are injected by developers.
3. **Headless Forwarding:** Every interactive atom utilizing Radix UI primitives must implement `React.forwardRef` and `asChild` composition to prevent hydration nesting errors (e.g. nested `<button>` tags).

## 4. Theme Token Subscriptions
Rather than using `rounded-md` or hardcoded heights, the L3 Atoms must subscribe to the global Inspector CSS variables defined by the theme parameters:
*   `--button-border-radius`
*   `--input-border-width`
*   `--toggle-border-radius`
*   `--slider-thumb-size`
*   `--accordion-height`

## 5. Mandatory Audit Grep Scans
The `zap-audit` process MUST flag any violation of the L3 structural contract.

```bash
# Sweep for developers hardcoding native buttons with raw padding/radius instead of importing the canonical L3 <Button>
grep -rE '<button className="[^"]*(p-[0-9]|rounded|bg-)[^"]*"' src/ --include="*.tsx"

# Sweep for generic input tags styling their own borders rather than using L3 <Input>
grep -rE '<input className="[^"]*(border|rounded)[^"]*"' src/ --include="*.tsx"

# Sweep for L3 Atoms missing the critical forwardRef implementation
grep -rl "export const Button =" src/genesis/atoms/ | xargs grep -L "forwardRef"
```

## 6. Resolution Guide for `zap-fix`
When instructed to repair an L3 violation:
1. Locate the legacy or inline DOM element (`<button className="...">`).
2. Replace it entirely with the canonical imported L3 atom (`import { Button } from '@/genesis/atoms/...'`).
3. Strip any spatial margin ghost classes (`mb-4`, `mt-2`) from the component itself, and instead wrap the components in a layout spacer `div` or use the parent's `gap`.
4. Ensure text casing inside interactive elements correctly inherits the L3 theme typography standard (e.g., removing hardcoded `.uppercase` if the theme demands `.normal-case`).
