# L1 Canvas Data Dictionary (Metro Theme)

> [!IMPORTANT]
> **ZAP-AUDIT DIRECTIVE:** This file is the absolute source of truth for the **L1 Canvas Layer**. The `zap-audit` skill must use this to verify the structural integrity of the application root.

## 1. Definition & Role
The **L1 Canvas** is the primary background layer of the application. It sits directly above L0 (Base) and acts as the infinite viewport or "wallpaper" that houses the application.

## 2. Structural Tokens
- **Mandatory Background:** `bg-layer-canvas`
- **Forbidden Overrides:** You cannot inject raw hex colors or secondary background tokens (like `bg-layer-cover`) onto an L1 tag.

## 3. Topographic Rules
1. **Viewport Bleed:** L1 MUST stretch to fill the viewport (`min-h-screen`, `h-full`, `w-full`).
2. **No Hard Boundary:** L1 does NOT use border radiuses or explicit outlines. It is infinite.
3. **Typography Inheritance:** L1 sets the baseline typography root (`text-on-surface`).
4. **Child Restrictions:** Because L1 is infinite, it should immediately host an **L2 Wrapper** or Layout Grid to contain the content.

## 4. Mandatory Audit Grep Scans
The `zap-audit` process MUST flag any violation of the L1 structural contract.

```bash
# Sweep for L1 nodes that are missing the mandatory token
grep -rE 'className="[^"]*min-h-screen[^"]*"' src/app --include="*.tsx" | grep -v 'bg-layer-canvas'

# Sweep for illegal border radiuses applied to the canvas
grep -rE 'class(Name)?="[^"]*bg-layer-canvas[^"]*rounded' src/ --include="*.tsx"
```

## 5. Resolution Guide for `zap-fix`
If an L1 layer fails the audit:
1. Strip any hardcoded `bg-slate-100` or raw hex values and enforce `bg-layer-canvas`.
2. Strip any `rounded-md` or `border` utilities. The Canvas does not have edges.
3. Ensure it possesses viewport-stretching geometry (`min-h-screen w-full`).
