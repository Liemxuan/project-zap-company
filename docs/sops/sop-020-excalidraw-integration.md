## 1. Objective

Provide an internal diagramming canvas within ZAP Design Engine at `/debug/metro/diagrams`, powered by `@excalidraw/excalidraw`. This eliminates the need to jump to external tools for architecture diagrams, flowcharts, and system maps. The canvas supports loading and exporting `.excalidraw` JSON files — including those generated by the `excalidraw-diagram` agent skill.

## 2. Infrastructure Setup

### 2.1 Install the Package

From the `zap-design` package root:

```bash
cd packages/zap-design
npm install @excalidraw/excalidraw
```

**Verify installation:**

```bash
grep excalidraw packages/zap-design/package.json
# Should show: "@excalidraw/excalidraw": "^0.x.x"
```

### 2.3 Key Technical Decisions

- **Dynamic import with `ssr: false`**: Excalidraw uses `<canvas>`, `window`, and `document` APIs. It MUST be loaded client-side only via `next/dynamic`.
- **No iframe**: The React component is embedded directly — full native integration.
- **File format**: `.excalidraw` files are plain JSON. No binary format, no external dependencies.
- **Zero backend**: No server-side storage. Files are loaded from disk and exported to disk.

## 3. Usage

### 3.1 Access the Canvas

1. Navigate to the ZAP Design Engine (default: `http://localhost:3002`)
2. In the left sidebar under **L1: Tokens**, click **Diagrams**
3. The Excalidraw canvas loads with full drawing tools

### 3.2 Load an Existing Diagram

1. Click the **Load** button in the toolbar
2. Select any `.excalidraw` or `.json` file
3. The canvas re-renders with the loaded scene

### 3.3 Export a Diagram

1. Click the **Export** button in the toolbar
2. The current scene is downloaded as a `.excalidraw` JSON file
3. This file can be reopened in: Excalidraw, VS Code (with extension), or Obsidian

### 3.4 Agent-Generated Diagrams

The `excalidraw-diagram` skill generates `.excalidraw` JSON files programmatically. Agents can create architecture diagrams, flowcharts, and system maps via natural language, then the files can be loaded directly into this canvas for review or editing.

**Agent workflow:**

1. User says: "create a diagram of the auth flow"
2. Agent generates `auth-flow.excalidraw` using the skill
3. User opens the Diagrams page and loads the file
4. User edits/refines the diagram visually
5. User exports the final version

## 4. Excalidraw Component API

The embedded `<Excalidraw />` component exposes an API via the `excalidrawAPI` ref:

| Method | Description |
|---|---|
| `getSceneElements()` | Returns all elements on the canvas |
| `getAppState()` | Returns current canvas state (zoom, background, etc.) |
| `getFiles()` | Returns any embedded binary files (images) |
| `updateScene({ elements, appState })` | Programmatically update the canvas |
| `resetScene()` | Clear everything |
| `scrollToContent()` | Auto-zoom to fit content |

## 5. Troubleshooting

### "ReferenceError: window is not defined"

Excalidraw was imported without `ssr: false`. Ensure the dynamic import uses:

```tsx
const Excalidraw = dynamic(
    async () => (await import('@excalidraw/excalidraw')).Excalidraw,
    { ssr: false }
);
```

### Canvas not rendering / blank page

- Check browser console for errors
- Ensure the parent container has explicit height (`h-full` or `100vh`)
- Excalidraw needs a sized container — it won't render in a zero-height div

### "Module not found: @excalidraw/excalidraw"

```bash
cd packages/zap-design && npm install @excalidraw/excalidraw
```
